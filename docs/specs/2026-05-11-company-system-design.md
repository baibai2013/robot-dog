# 仿生机器狗公司系统 — 设计文档

**日期:** 2026-05-11  
**状态:** MVP 阶段  
**范围:** 公司系统 + robot-dog 项目框架

---

## 1. 整体结构

两个独立文件夹，平级关系：

```
~/work/
├── company/          # 公司操作系统（基础设施、员工定义、通信）
└── projects/
    └── robot-dog/    # 仿生机器狗项目（CAD/PCB/固件/仿真）
```

`company/` 是可复用的公司底座，未来可以承载其他项目。  
`projects/robot-dog/` 是独立项目，只包含机器狗相关内容。

## 2. 公司组织架构

共 9 名 AI 员工，分两层：

### 管理层（3人）
| 角色 | 职责 |
|---|---|
| 产品经理 | 产品需求、规格定义、用户场景 |
| 项目经理 | 排期、里程碑推进、跨部门协调 |
| 技术负责人 | 技术决策、架构把关、跨域技术协调 |

### 工程层（6人）
| 角色 | 职责 |
|---|---|
| 机械工程师 | 结构设计、CAD（build123d）、样机规格 |
| 硬件工程师 | 原理图、PCB、元器件选型 |
| 固件工程师 | 嵌入式代码、通信协议、电机控制 |
| 算法工程师 | 步态、IK/FK、仿真（PyBullet） |
| 测试工程师 | 集成验证、一致性检查、测试报告 |
| 成本工程师 | BOM 成本核算、元器件比价、整机预算 |

**你的角色:** CEO / 技术总监 — 只看进度、给方向、审批关键节点。

**双模式:**
- **聊天下指令** — 在部门频道 @ 员工分配任务

### 系统频道说明
| 频道 | 内容 | 谁写 |
|---|---|---|
| `#状态` | 进度卡片、任务完成通知、系统日志 | 机器人自动推送 |
| `#待审批` | 需要你决策的 gate 任务 | 机器人触发 |
| `#文件输出` | CAD 文件、报告、原理图链接 | 员工完成时自动归档 |

## 5. 技术选型（本地自托管）

| 组件 | 工具 | 用途 |
|---|---|---|
| 聊天 + 频道 | Mattermost（Docker） | 员工通信、你下指令 |
| 代码 + 文件版本 | Gitea（Docker） | 输出物存储、PR 审批 |
| 自动化枢纽 | n8n（Docker） | Mattermost → Agent → Gitea 串联 |
| 数据库 | PostgreSQL（Docker） | 三个服务共用 |
| AI 大脑 | Claude API | 所有员工的推理核心 |
| 看板 UI | 自建 HTML | 三栏界面，读 state.yaml |

---

## 6. 核心流程

```
你在频道输入指令
      ↓
n8n 接收 Mattermost webhook
      ↓
调用对应 Claude Agent（员工）
      ↓
Agent 工作 → 输出提交到 Gitea
      ↓
n8n 推送结果卡片到 #状态频道
      ↓
如果是 Gate → 推送到 #待审批，等你回复
      ↓
你审批 → 解锁下游任务
```

## 7. MVP 范围

**包含:**
- [ ] `company/` 文件夹结构 + 9 个员工定义
- [ ] `projects/robot-dog/` 迁移重组
- [ ] Mattermost + Gitea + n8n Docker 一键启动
- [ ] 三栏 HTML 看板（左导航 + 中聊天 + 右任务面板）
- [ ] n8n 基础工作流：Mattermost 指令 → Agent → #状态推送

**不包含（迭代）:**
- 移动端适配
- 多项目切换
- 员工"记忆"跨任务持久化
- 实时语音指令

---

## 8. 成功标准（MVP）

1. 一条 `docker compose up` 把所有服务跑起来
2. 你在 Mattermost 发一条指令，机械工程师 Agent 能响应并回报结果
3. 任务状态自动同步到右侧任务面板
4. Gate 任务在 `#待审批` 正确弹出
